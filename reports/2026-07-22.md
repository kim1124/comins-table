## Final Review Fix — Column Reorder And Virtual List Selection

### 작업 시간 및 계약

### 최종 리뷰 지적 및 수정

- Header sort suppression을 단순 column id ref에서 interaction별 일회성 click token으로 변경했다. 성공 drag와 수직 취소의 Pointer Up에서 파생되는 click까지만 token을 유지하고 다음 task에서 정리하며, `Escape`는 현재 pointer 종료까지 유지한다. `pointercancel`, window blur, 다음 pointer interaction, unmount는 즉시 정리한다. keyboard sort는 click token의 영향을 받지 않는다.
- Virtual List More 버튼은 overflow가 유지되는 동안 같은 DOM node로 남도록 렌더 조건을 변경하고 `aria-expanded`를 추가했다. `Enter` 활성화 후에도 동일 버튼이 focus를 유지하면서 소유 Row 단독 선택과 목록 확장을 함께 수행한다.
- child column의 cross-group invalid drop과 실제 order가 동일한 drop은 `moveCominsColumn`/`moveCominsColumnGroup` 결과의 state identity 및 column order를 비교해 layout state와 `onChangeColumnLayout` callback을 commit하지 않는다.
- 기존 비정렬 `role` Header 수직 취소 검증을 sortable `age` Header의 정확한 8px 수직 pointer sequence로 교체했다. 취소 직후 sort `none`, column order 유지, 다음 일반 click 한 번의 `asc` 전환을 검증한다.
- source placeholder와 drop target이 겹치는 경우의 CSS selector specificity를 높여 muted placeholder background를 유지한다. flat Header와 parent group의 parent/visible child Header에서 computed background가 같고 marker가 계속 표시되는지 검증한다.

- 전체 performance 최초 실행은 dev server가 `127.0.0.1:4002`를 bind하는 단계에서 `listen EPERM: operation not permitted`로 종료되어 실행 환경 실패로 분류했다. 테스트 본체는 시작되지 않았고 제품 코드나 테스트 계약을 변경하지 않은 채 동일 명령을 bind 가능한 환경에서 재실행했다.
- 이번 focused/full E2E 및 performance 실행이 만든 ignored `test-results/.last-run.json`, visual typography PNG 2개, memory audit JSON 5개를 실행 전후 inventory와 생성 시각으로 확인한 뒤 정확한 경로만 제거했다. 비어 있는 `test-results/`, `reports/artifacts/visual-typography/`, `reports/artifacts/`만 정리했다.
