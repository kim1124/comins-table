- physical scrollbar 성능 테스트가 checked component checkbox의 data URI SVG를 Chromium 내부 문서로 최초 로드한 뒤에도 cold Basic 문서 수와 정확히 같아야 한다고 비교한 테스트 계약 문제를 수정했다.

- PRODUCT LEAK 아님: Basic 복귀 후 frame은 1개, 실제 element 수는 610개로 원복되며, 반복 방문 후에도 document 수는 2에서 증가하지 않았다.
- 기존 node와 listener 임계값 및 별도 physical-scrollbar 응답성 테스트는 변경 전에도 통과했다.

### 변경 파일

- `test/playwright/specs/physical-scrollbar.spec.ts`
- `reports/2026-07-22.md`

### 검증

- RED: `npm run test:perf -- test/playwright/specs/physical-scrollbar.spec.ts --workers=1`에서 recovery 테스트가 `Expected: 1`, `Received: 2`로 실패했고 responsiveness 테스트는 통과했다.
- PASS: post-load document 기준 적용 후 동일 명령의 2개 테스트가 모두 통과했다.
