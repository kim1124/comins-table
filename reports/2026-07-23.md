# 2026-07-23 Multi-column Sort

## 작업 일시

- 2026-07-23 10:24–10:49 KST
- 적용 계약: `COMINS_CONTRACT` v1.2

## 요약

- 기존 단일 정렬 API를 유지하면서 우선순위 기반 `CominsSortModel`을 추가했다.
- `multiSort`를 명시한 경우 `Shift`+click 또는 `Shift`+`Enter`/`Space`로 조건을 추가·변경·제거하고, 일반 조작은 기존 단일 정렬을 유지한다.
- Flat Table과 Tree Grid sibling 정렬, 2Depth child Header, custom comparator, stable tie, Ref 복원, callback 관찰, 컬럼 제거 정규화를 같은 모델에 연결했다.
- Header에 방향 아이콘과 1-based 우선순위 badge를 표시하고, 첫 조건만 `aria-sort`를 사용하며 후속 조건은 숨김 상태 설명으로 보완했다.
- README, 영어·한국어 사용자 문서, API/Ref 예제, Header Playground, 옵션 가이드, changelog에 기능을 명시했다.
- 공개 npm `latest=0.1.3`을 확인하고 release candidate version을 `0.1.4`로 정렬했다.
- 원격 push, publish, tag, GitHub Release는 아직 수행하지 않았다.

## RED / 수정 근거

- 최초 focused Vitest는 `setCominsSortModel`, Ref 모델 API, 모델 callback이 없어 5건 실패하고 66건 통과했다.
- 컬럼 제거 회귀 테스트는 내부 모델은 정규화됐지만 `onChangeSort`가 호출되지 않아 1건 실패했다. Column 정의 동기화 경로에서 정규화된 단일 projection과 전체 모델을 각각 한 번 통지하도록 수정했다.
- 최초 `npm run verify`는 README 목차 고정 및 기존 Ref 문구 계약 2건에서 실패했다. 새 기능을 기존 `Controlled Model`의 하위 섹션으로 배치하고 단일 정렬 문구를 보존해 문서 계약을 수정했다.
- 최초 10만 Row focused performance 테스트는 Body viewport 내부에서 Header를 찾는 잘못된 locator로 timeout이 발생했다. 상위 `.comins-table` 범위로 locator를 교정했다.

- 서버 정렬 모델, always-accumulate click, Ctrl/Cmd trigger, 최대 조건 수 prop, initial controlled `sortModel` prop은 이번 범위에 포함하지 않았다.
