### Root Cause Classification

- EXECUTION ENVIRONMENT / IMPLEMENTATION INTERACTION: GitHub Actions run `29752914758` failed twice with exit code `134` and glibc heap corruption while the hygiene scanner processed reachable history.
- INSUFFICIENT FIRST MITIGATION: limiting content requests to `128` objects passed local Linux reproduction but run `29754112830` still aborted with `malloc(): invalid next size (unsorted)` while using synchronous binary-output capture.
- INSUFFICIENT SECOND MITIGATION: streamed content parsing passed Node.js 24.14.0 reproduction but run `29755057113` still aborted because the hosted runner had advanced to Node.js 24.18.0.
- EXACT REPRODUCTION: x86_64 Ubuntu 24.04, glibc 2.39, and Node.js 24.18.0 reproduced exit code `134` after the scanner printed its successful `1,024`-object result.
- CONFIRMED TRIGGER: history streaming without decoding passed; UTF-8 decoding passed; direct UTF-16LE decoding of reachable odd-length object buffers reproduced heap corruption.
- CONFIRMED FIX BOUNDARY: truncating only the incomplete trailing byte before UTF-16LE decoding preserved detection and completed the identical `1,024`-object scan with exit code `0` in the exact reproduction container.
